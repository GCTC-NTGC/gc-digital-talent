# Dialog

> A window overlaid on either the primary window or another dialog window, rendering the content underneath inert.

This is a styled version of [@radix-ui/dialog](https://www.radix-ui.com/docs/primitives/components/dialog). The exception being the `Header` and `Footer` which are custom components for styling purposes.

## Subcomponents

- `Dialog.Root`
- `Dialog.Close`
- `Dialog.Content`
- `Dialog.Trigger`
- `Dialog.Header`
- `Dialog.Footer`

## Usage

<Story id="components-dialog--not-open" />

```tsx
import Dialog from "@gc-digital-talent/ui/Dialog";
import Button from "@gc-digital-talent/ui/Button";

const ExampleDialog = () => (
  <Dialog.Root>
    <Dialog.Trigger>
      <Button>Open Dialog</Button>
    </Dialog.Trigger>
    <Dialog.Content>
      <Dialog.Header subtitle="short description">
        Dialog Title
      </Dialog>
      <p>Dialog body</p>
      <Dialog.Footer>
        <Dialog.Close>
          <Button>Close</Button>
        </Dialog.Close>
      </Dialog.Footer>
    </Dialog.Content>
  </Dialog.Root>
);
```

## Props

### Header

```tsx
export interface DialogHeaderProps {
  subtitle?: React.ReactNode;
  children: React.ReactNode;
  color: Color;
}
```

- `subtitle`: Appears directly below the title and acts a description
  - See: [WAI-ARIA Roles, States, and Properties](https://www.w3.org/WAI/ARIA/apg/patterns/dialogmodal/#:~:text=Optionally%2C%20the%20aria,single%20unbroken%20string.)
- `children`: The title of the `Dialog`
- `color`: Hydrogen theme colour for the `Header` background
